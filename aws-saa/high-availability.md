# High Availability

Aims to **ensure** an agreed level of operational **performance**, usually
**uptime**, for a **higher than normal period**.

It is not:

- a means to stop failure at all cost
- having no outages
- about user experience (user needing to log in again is ok)

It is about being able to get back to an operational state as fast or automated 
as possible.
